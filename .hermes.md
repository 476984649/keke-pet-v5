7.2 跨模块通信（EventBus 标准）
❌ 禁止：模块间直接 import 函数造成循环依赖。

✅ 强制：使用 modules/event-bus.js 作为唯一通信桥梁。

javascript
// src/modules/event-bus.js
class EventBus extends EventTarget {
    emit(eventName, detail) {
        this.dispatchEvent(new CustomEvent(eventName, { detail }));
    }
    on(eventName, handler) {
        this.addEventListener(eventName, handler);
    }
    off(eventName, handler) {
        this.removeEventListener(eventName, handler);
    }
}
export const bus = new EventBus();
使用示例：

javascript
// state.js — 数据变更时发布事件
import { bus } from './event-bus.js';
export function setAffection(val) {
    affection = val;
    bus.emit('affection:updated', { value: val });
}

// renderer.js — 监听事件并响应
import { bus } from './event-bus.js';
bus.on('affection:updated', (e) => {
    updateHeartDisplay(e.detail.value);
});

第 11 章 · 编译错误处理与架构保护机制
本章旨在防止 Agent 在遇到编译错误时，采取“最小代价修复”策略（如删除依赖、注释代码、绕过功能），从而破坏项目架构完整性。

11.1 编译错误处理原则（优先级排序）
当遇到编译错误时，Agent 必须按以下优先级顺序尝试修复：

优先级	修复方向	说明
1	修复自己的代码	检查是否是自己写的代码存在类型错误、生命周期问题、模块导入路径错误、函数签名不匹配等。
2	调整依赖版本	如果错误涉及依赖版本不兼容，优先尝试升级/降级依赖版本，而非删除功能。
3	适配 API 变更	如果错误涉及 API 变动（如 Tauri v1 → v2 的 breaking changes），优先查阅官方迁移指南，适配新 API。
4	补充缺失的依赖	如果错误提示“找不到某个 crate”或“feature 不存在”，优先添加正确的依赖或 feature，而非删除引用。
5	报告并等待决策	如果以上 4 种方式均无法解决，或触及“架构红线”（见 11.2 节），必须暂停并汇报，等待用户指令。
