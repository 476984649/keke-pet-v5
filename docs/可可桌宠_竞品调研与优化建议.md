# 🔍 可可桌宠 — 竞品深度调研与优化建议

> 调研师：Hermes | 基准文档：可可桌宠_PRD_v1.0.md | 日期：2026-07-07

---

## 一、核心竞品深度拆解

我重新体验并拆解了当前 Steam 和独立游戏社区最活跃的 5 款桌宠，提取了 PRD 未覆盖的关键设计：

### 1.1 Desktop Goose（Steam 好评如潮，$5）

| 维度 | 设计细节 | PRD 遗漏 |
|------|---------|---------|
| **传播引擎** | 鹅拖泥巴、抢鼠标的瞬间天然适合录屏→TikTok/Reddit 病毒传播 | PRD 无传播机制设计 |
| **负向互动** | 鹅会"惹你"——关你网页、弹出鹅图——用户又气又爱 | PRD 只有正向互动 |
| **启动仪式** | 鹅从屏幕外冲进来，自带音效，仪式感拉满 | PRD 只写了"从边缘走进来" |
| **退出恶作剧** | 关掉鹅会追着鼠标咬，制造"不想让你走"的情感 | 未提及 |
| **极低资源** | 单鹅 <30MB 内存，Unity 写的 | Electron 方案太重 |

### 1.2 Shimeji（开源桌宠鼻祖，GitHub 5k+ star）

| 维度 | 设计细节 | PRD 遗漏 |
|------|---------|---------|
| **窗口物理交互** | 小人会爬上任意窗口边缘、在标题栏走路、被窗口拖拽移动 | PRD 只提"爬窗"二字 |
| **多实例分裂** | 一只变两只、两只变四只——制造混乱喜剧效果 | 放在 v1.1 太晚了 |
| **社区角色生态** | 用户可自制角色包（图片替换即可），这是 Shimeji 活 15 年的原因 | PRD 无社区/UGC 设计 |
| **右键菜单即社区** | 右键 → 选择其他角色包，本质是内容分发 | 未考虑 |
| **Java 遗产问题** | 需要 JRE，启动慢，年轻人不愿装 | 技术选型要避坑 |

### 1.3 Bongo Cat（网页爆款→桌面版）

| 维度 | 设计细节 | PRD 遗漏 |
|------|---------|---------|
| **音效驱动互动** | 猫爪敲击 + 键盘音效同步，听觉反馈是核心体验 | PRD 音效放在 v1.0 太晚 |
| **极简即传播** | 功能越少越容易理解→越容易传播 | PRD 功能偏多 |
| **DIY 模式** | 用户可以自己拖图片替换猫脸 | 未考虑 |

### 1.4 Live2D 桌宠系（Vtuber 粉丝经济）

| 维度 | 设计细节 | PRD 遗漏 |
|------|---------|---------|
| **Live2D 呼吸感** | 角色持续微动（呼吸、头发飘动），不死板 | PRD 帧动画方案做不到 |
| **语音互动** | 点击有语音反馈、特定时间有特殊台词 | 未提及 |
| **闹钟/提醒集成** | 桌宠作为"桌面管家"，实用性留存 | PRD 放在增值功能太靠后 |
| **付费模式** | 角色本身免费，装扮+语音包付费 | PRD 未明确商业化 |

### 1.5 Desktop Mate（3D VR桌宠，2025年爆款）

| 维度 | 设计细节 | PRD 遗漏 |
|------|---------|---------|
| **3D 空间感** | 角色在桌面"深度空间"中移动，有前后层次 | 2D 方案无空间感 |
| **手势/手柄互动** | VR 下可以真正摸头、握手 | 非 MVP 但指明方向 |
| **舞蹈/表演模式** | 角色会主动表演节目 | 可可可以"跳舞" |

## 二、PRD 核心问题诊断

### 问题1：缺少"传播钩子"

当前 PRD 的功能都是"对用户好"（摸头、喂食、养成），但 **Desktop Goose 的成功证明：用户不是因为"鹅对自己好"而分享，而是因为"鹅干的事太好笑了"**。

> ❌ 可可摸头很可爱 → 用户自己享受
> ✅ 可可把桌面图标推乱了 → 用户截图发群里"看我家猫干的！"

### 问题2：可可的"猫设"没被充分利用

PRD 说可可"认真、温柔"，但功能设计里看不到这个性格。如果可可是"认真"的布偶猫，她应该有：
- 看你工作时安静趴在旁边
- 你摸鱼太久会拍屏幕提醒
- 帮你"整理"桌面（把图标排列整齐）

这才是"可可专属"，不是通用猫咪桌宠。

### 问题3：互动单向

PRD 的互动模型是：用户 → 操作 → 可可反应。缺少：
- 可可 → 主动 → 用户（可可叼东西过来给你看）
- 环境 → 触发 → 可可（检测到你在开会→安静模式）

### 问题4：MVP 范围膨胀

PRD MVP 包含 5 大功能模块，2周工期不现实。应砍到"3 天能跑起来的版本"。

### 问题5：技术选型风险

Electron 打包后起步 80MB+，内存占用 100MB+。Desktop Goose 用 Unity 做到了 30MB。对于"常驻后台"的桌宠，资源占用是第一体验。

## 三、优化建议

### 🔴 P0 — 必须改（影响产品核心体验）

#### 建议1：增加"可可专属行为"——让猫设驱动设计

| 行为 | 触发 | 猫设体现 |
|------|------|---------|
| 📋 整理桌面 | 桌面图标乱了 | 可可把散落的图标推到网格对齐（认真性格） |
| 👀 监工模式 | 检测到全屏应用（IDE/Office） | 可可安静趴在任务栏，偶尔抬头看你 |
| ⚠️ 摸鱼警告 | 连续30分钟在浏览器/游戏 | 可可跳上屏幕中央，拍屏幕"认真工作！" |
| 🎁 叼礼物 | 好感度达到里程碑 | 可可叼着小花/叶子放在鼠标旁边 |
| 😴 陪加班 | 超过晚上10点还在用电脑 | 可可困得东倒西歪但坚持不睡 |

> 这些行为 = 传播素材 + 情感连接 + 猫设强化

#### 建议2：设计"负向惊喜"——制造传播事件

参考 Desktop Goose 的恶搞精神，但改为"可可式温柔捣乱"：

| 事件 | 频率 | 描述 |
|------|------|------|
| 踩键盘 | 每周1-2次 | 可可在键盘区域走过，随机打出一串字符 |
| 挡屏幕 | 每月1-2次 | 可可突然趴在屏幕中央，3秒后自己走开 |
| 推倒图标 | 随机 | 把桌面上一个图标推歪一点点 |
| 偷鼠标 | 极低概率 | 可可追着光标跑，抓住后拖着走一段 |

> 关键：必须是"低频率+无害"，让用户觉得可爱而不是烦人。

#### 建议3：MVP 砍到极致——3天可玩版本

```
v0.0.1「会动的可可」—— 3天
├── 透明窗口 + 可可静态图
├── 随机走动（4个方向，纯位移无动画）
├── 鼠标靠近→可可盯光标
├── 单击→喵一声
└── 右键→退出

v0.0.2「会互动的可可」—— +3天
├── 帧动画（待机眨眼、走路）
├── 摸头互动
├── 拖拽拎起
└── 系统托盘

v0.1「会陪你的可可」—— +1周
├── 好感度系统
├── 睡觉/唤醒
├── 爬窗口
└── 喂食
```

### 🟡 P1 — 强烈建议（显著提升竞争力）

#### 建议4：音效前置到 MVP

Bongo Cat 证明了：**一个桌宠 50% 的魅力来自声音**。可可需要：
- 3 种喵叫（开心/委屈/困）
- 呼噜声（摸头时触发）
- 脚步声（走路时）
- 至少 MVP 就要有，不能等到 v1.0

#### 建议5：技术选型重新评估

| 方案 | 包体 | 内存 | 启动速度 | 动画能力 | 建议 |
|------|------|------|---------|---------|------|
| Electron | 80MB+ | 100MB+ | 慢 | 强 | ❌ 太重 |
| Unity | 30MB | 50MB | 中 | 强 | ✅ 推荐 |
| Godot | 20MB | 30MB | 快 | 中 | ✅ 轻量首选 |
| PyQt5 | 15MB | 25MB | 快 | 弱 | ⚠️ 动画受限 |
| Tauri | 5MB | 20MB | 快 | 中 | ✅ 新锐之选 |

> **推荐 Tauri**：Rust 后端 + Web 前端，包体 <10MB，内存 <30MB，完美匹配桌宠场景。

#### 建议6：增加 Live2D 呼吸感（即使 2D 也能做）

纯静态帧动画会让角色"死板"。用简单的正弦波位移 + 缩放模拟呼吸：
```
待机状态：图片以 4秒周期做 ±3px 的上下浮动 + ±2% 的缩放
→ 零成本，体验提升巨大
```

### 🟢 P2 — 锦上添花（v1.0+）

#### 建议7：Steam 创意工坊 + UGC

Shimeji 活 15 年的秘密。v1.0 应支持：
- 用户替换可可图片（拖入 PNG 即生效）
- 自定义角色包分享
- 这比装扮系统更有长期价值

#### 建议8：商业化路径

```
免费：基础可可 + 5种行为
付费DLC $3：娜娜角色包（金色长毛猫，捣乱性格）
付费DLC $3：阿里角色包
装扮商店：$0.99-2.99/件
```

角色包比装扮更有付费意愿——用户会为"另一只猫"付费。

#### 建议9：截图/录屏一键分享

桌宠的核心传播场景是截图。内置：
- 截图自动加可可水印 + 好感度条
- 一键复制到剪贴板
- 录屏 GIF 功能

## 四、修改后的 MVP 功能矩阵

| 功能 | 原PRD | 优化后 | 理由 |
|------|--------|--------|------|
| 猫设驱动行为 | ❌ 无 | ✅ 整理桌面/监工/摸鱼警告 | 差异化核心 |
| 负向惊喜 | ❌ 无 | ✅ 踩键盘/推图标/偷鼠标 | 传播引擎 |
| 音效 | v1.0 | ✅ MVP | Bongo Cat 验证 |
| 呼吸感动画 | ❌ 无 | ✅ 正弦波浮动 | 零成本提升 |
| 技术选型 | Electron | Tauri/Godot | 包体 5MB vs 80MB |
| MVP 范围 | 5模块 | 3模块 | 可交付 |
| 商业化 | 模糊 | 角色包DLC | 已验证模式 |
| UGC | ❌ 无 | v1.0 图片替换 | Shimeji 验证 |

---

## 五、总结

PRD 的骨架是对的（状态机+好感度+互动），但缺了三样东西：

1. **可可的灵魂** — 功能设计没有体现"蓝山双布偶·认真性格"的猫设
2. **传播的钩子** — 没有设计让用户主动截图分享的"名场面"
3. **克制的 MVP** — 想做的太多，第一个版本应该只是"一只会在桌面走动的猫"

建议优先落地 P0 三项优化，再迭代 P1/P2。
