## 一、设计定位

### 1.1 设计关键词

```
温  暖  治  愈  简  洁  俏  皮
```

- **2D 扁平风格**，圆角为主，低饱和度配色
- 可可本身是写实风蓝山双布偶 → UI 采用**轻量毛绒感**呼应
- 切忌花哨——可可才是主角，UI 是配角

### 1.2 色彩系统

```
主色：

  🩵 可可蓝  #7EB5E0    — 主题色，取自可可蓝眼睛
  🤍 奶油白  #FFF5EE    — 背景色，取自可可白毛
  💛 暖阳黄  #F5D76E    — 点缀色，互动反馈

辅助色：

  🩶 毛灰    #B8B8B8    — 不可用状态、边框
  🖤 深灰    #3A3A3A    — 正文文字
  ⚪ 纯白    #FFFFFF    — 卡片背景

语义色：

  ❤️ 珊瑚红  #FF6B6B    — 好感度、危险操作
  💚 叶绿    #7EC8A0    — 成功、完成
  🧡 杏色    #FFB87A    — 提醒、警告

渐变：

  🌅 日出     #FFE8CC → #FFB87A  — 好感度高时氛围
  🌙 月光     #7EB5E0 → #4A90D9  — 夜间模式
```

## 三、各模块详细设计

### M1 — 可可主窗口

```
┌──────────────────────────────────────────────┐
│    (透明)                                       │
│           ◀ 可可在桌面活动区域 (150×200px)       │
│                                               │
│  ┌────────────────┐                           │
│  │   (可可主体)   │   ← 透明背景，只有可可本体   │
│  │   帧动画播放    │                            │
│  │   鼠标互动区域  │                            │
│  └────────────────┘                           │
│                                               │
│  窗口特性：                                     │
│  • 透明通道 (transparent=true)                  │
│  • 置顶 (alwaysOnTop=true)                      │
│  • 无边框 (frame=false, decorations=false)      │
│  • 大小: 默认 150×200px, 可缩放 (120-300px)    │
│  • 点击穿透: 仅互动模式截获鼠标事件             │
└──────────────────────────────────────────────┘
```

**交互说明**：
- 默认情况下鼠标可以"穿过"可可区域点击桌面
- 只有可可主动走到图标/窗口上时，才拦截事件
- 鼠标在可可身上悬浮 → 可可抬头看光标
- 按住可可拖动 → 拎起模式

### M2 — 右键菜单

```
                        ┌──────────────────────┐
                        │  🐱 可可 — 好感 ██░░ 72│  ← 顶部信息栏
                        │──────────────────────│
                        │  🐟 喂食              │
                        │  ── 小鱼干  · 猫罐头 ·│  ← 子菜单展开
                        │  🎀 装扮              │
                        │  📊 状态              │
                        │  📔 日记              │
                        │  🎬 录制名场面        │  ← 新功能
                        │  😊 表情包工坊        │  ← 新功能
                        │──────────────────────│
                        │  ⚙️ 设置              │
                        │  ❌ 退出              │
                        └──────────────────────┘

  尺寸: 200×320px  |  背景: #FFFFFF  |  圆角: 12px
  阴影: 0 4px 16px rgba(0,0,0,0.15)
  每个菜单项: height 36px, hover 背景 #FFF5EE
```

**交互细节**：
- 鼠标悬停 200ms 后展开子菜单（防误触）
- 点击菜单项 → 菜单收起 → 触发对应功能
- 点击菜单外区域 → 菜单关闭
- 喂食子菜单展开时显示食物列表，每个食物有图标+名称+冷却状态

### M3 — 可可气泡

```
           ┌──────────────────┐
           │ 喵~ 今天工作累了吗│
           │  (ฅ´ω`ฅ)        │
           └───────┬──────────┘
                   │
               ▼ 可可
```

| 气泡类型 | 背景色 | 示例文案 |
|---------|-------|---------|
| 开心 | 主蓝 #7EB5E0 | "贴贴~" |
| 困倦 | 暖黄 #F5D76E | "好困哦 zzz" |
| 委屈 | 粉 #FFB8C6 | "呜...你拽疼我了" |
| 惊喜 | 渐变日出 | "小鱼干！！" |
| 日常 | 主蓝 70% | "现在几点了呀？" |

**规格**：
- 最大宽度 200px
- 背景圆角 16px
- 箭头向下 8px
- 显示时长：3-6秒自动消失
- 每 30 秒最多出现一次

## 五、Tauri 实现要点

### 5.1 窗口配置

```json
{
  "windows": [
    {
      "title": "可可桌宠",
      "label": "keke-pet",
      "url": "/index.html",
      "width": 150,
      "height": 200,
      "resizable": true,
      "minWidth": 120,
      "minHeight": 160,
      "transparent": true,
      "decorations": false,
      "alwaysOnTop": true,
      "visibleOnAllMonitors": false
    }
  ]
}
```

### 5.2 右键菜单

```json
// 右键菜单 → 系统原生ContextMenu 或 自定义HTML弹层
// 推荐：用 Tauri Shell 创建原生菜单 或 纯CSS弹层
// 原则：菜单不跟随可可移动，在点击位置展开
```

### 5.3 动画实现

```
// 可可帧动画：requestAnimationFrame + Canvas
// 呼吸浮动：CSS transform + @keyframes (4s周期)
// 气泡：CSS opacity + transform 过渡
// 粒子特效：Canvas 独立层
```

## 六、低保真线框图参考

### 6.1 主交互流程

```
启动 → 可可从右走入 → 待机(呼吸+眨眼)
  └─→ 右键 → 菜单弹出
   ├─→ 喂食 → 子菜单 → 选择食物 → 可可吃 → 菜单关闭
   ├─→ 装扮 → 面板弹出 → 选择 → 穿戴 → 面板关闭
   ├─→ 状态 → 面板弹出 → 查看 → 点击外部关闭
   ├─→ 日记 → 面板弹出 → 阅读 → 关闭/分享
   ├─→ 录制 → 5秒倒计时 → 录制中 → 导出GIF
   ├─→ 表情 → 工坊弹出 → 制作 → 保存/分享
   └─→ 设置 → 面板弹出 → 调整 → 保存
```

### 6.2 窗口层级

```
z-index 9999: 可可主窗口（alwaysOnTop）
z-index 10000: 右键菜单
z-index 10001: 可可气泡
z-index 10002: 各面板（状态/日记/装扮等）
z-index 10003: 系统提示（节日彩蛋、通知）
z-index 负层: 桌面/其他应用
```
